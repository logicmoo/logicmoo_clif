Module prooftask_cm
Using the CM prover for proving tasks.
cmpr(+FormulaSpec,+Options)
Perform a proof task with CM, including preprocessing, compilation and prover invocation. Succeeds once or multiply, depending on the timeout option.
+FormulaSpec
Specifies the input formula. Possible values:
An atom
Interpreted as name of a file in TPTP FOF or CNF format. If the file does not exist, interpreted as TPTP (FOF or CNF) problem name.
A formula
Must be an implication or reverse implication.
A matrix
A first-order matrix.
+Options
A list of Key=Value pairs. The options are passed trough to preprocessing predicates. The following keys are recognized:
add_cm_options
See explanation of determining effective CM/PR options below. Can be specified multiply. Possible values:
CMOptions
CM options to add.
CMOptions1-CMOptions2
Pair of CM options to add and to remove.
add_pr_options
See explanation of determining effective CM/PR options below. Can be specified multiply. Possible values:
PROptions
PR options to add.
PROptions1-PROptions2
Pair of PR options to add and to remove.
cm_cfg
See explanation of determining effective CM/PR options below.
ConfigId
Identifier of a preconfigured CM option set
cm_options
See explanation of determining effective CM/PR options below. Value:
CMOptions
CM options.
cnf_trafo
Specifies CNF transformation to be used if the input problem is not in CNF. Default is attempt of standard CNF transformation with inference limit, else performing Plaisted-Greenbaum transformation. Possible values:
std
Standard CNF transformation.
pg
Plaisted-Greenbaum transformation.
pgi
Plaisted-Greenbaum transformation followed by identification of literals with detected equivalent non-recursive semi-definitions.
miniscope
Standard CNF with miniscoping (old implementation, not much tested).
auto(Limit)
Attempt standard CNF transformation in specified inference limit, else perform Plaisted-Greenbaum transformation.
eq
Specifies equality transformation applied if equality (=/2) is present in the input after conversion to matrix form. In the following Subs, Ref, Sym, Trans stand for substitutivity, reflexivity, symmetry and transitivity axioms, Symtrans and Transsym for combinations of symmetry and transitivity in a single axiom, and Flatten for a term flattening conversion.
std
Subs + Ref + Sym + Trans.
std_r_st
Subs + Ref + Symtrans (default).
std_r_ts
Subs + Ref + Transsym.
flatten
Flatten + Ref + Symtrans.
none
No special processing equality.
file
Specifies that the CM compiler writes a file that is then loaded, instead of using Prolog's internal mechanisms for loading compilation result. Useful to inspect the compiler output. YAP prolog requires this option.
Filename
Name of the file.
flatten
Apply a flattening matrix transformation, also in case wherethere is no equality present in the input. Possible values are (see key eq for meaning of the shorthands) [not fully tested]:
rst
Flatten + Ref + Symtrans.
r
Flatten + Ref.
plain
Flatten.
goal
Used to specify the goal clauses.
goal_clause_sign
Used to specify the goal clauses.
goal_clauses
Used to specify the goal clauses.
goal_pick
Used to specify the goal clauses.
info_progress
Control terminal output that indicates operation progress.
0
Disable terminal output indicating progress.
1
Enable terminal output indicating progress (default).
keep_gensyms
Control reinitialization of generated symbol names.
true
Do not reinitialize symbol generation, thus preventing name clashes with previously generated symbols.
false
Reinitialize symbol generation (default).
lean_cuts
Insert cuts in specific ways, leading to incompleteness (in part experimental).
none
No cut insertion (default).
CutSpecifier
Insert cuts as specified - see source code and examples.
mode
Special prover invocation modes for debugging.
default
Standard invocation.
rerun
For use with the file option. Do not create a new file with the compilation result.Instead load the existing file (with possible manual edits) and start the CM runtime system. Otherwise the execution is identical as in default mode (preprocessing etc. are also performed).
pr_cfg
See explanation of determining effective CM/PR options below.
ConfigId
Identifier of a preconfigured PR option set
pr_options
See explanation of determining effective CM/PR options below. Value:
PROptions
PR options.
proof
Specifies that proof terms are constructed (which costs runtime) and how they are output. See ... for the syntax specification of the proof terms.
-Variable
Bind the variable to the proof term.
writeq
Write the proof term with writeq.
write_canonical
Write the proof term with write_canonical.
pp
Pretty print the proof term.
writex
Write the proof term similarly to write_canonical, but with another implemenation that can handle very large terms.
result
Output the result of the proving task.
-Variable
Unified with an atom indicating the result: proved, failed_exhausted, or timeout.
simp
Specify simplifications that are performed separately on the matrices for the theorem and for the axioms.
Simplifications
A list of predefined terms that specify simplifications. default (as of May 2016): [sort_lits, condense(1000), subs(1000), split].
simp_prepare
Specify simplifications that are performed after the matrices for the theorem and for the axioms have been merged.
Simplifications
A list of predefined terms that specify simplifications. default (as of May 2016): [purity, elim_cheap].
tabx_dotgraph
Return or write a graph representation of the found proof in TABX format.
-Variable
The variable will be bound to a dotgraph term (see module dotgraph).
Filename
A GIF file with the graph will be written (requires installation of dot from GraphViz).
none
No graph output (default).
timeout
Specifies the mode of success and a possible timeout.
0
No timeout, solutions (found proofs) are enumerated. If there is no (more) proof found, the value of the result option is set to failed_exhausted.
Seconds
The specified timeout must be a number >0. There is a single solution, with result indicated by the value of theresult option.
Determining effective CM/PR Options. CM takes two lists of options, CMOptions that control compilation and PROptions that control the runtime. cm_options and pr_options specify these explicitly. cm_cfg and pr_cfg specify a label (atom or term) that is used to choose preconfigured option sets in dependency of matrix properties (Horn/non-Horn, number of clauses). add_cm_options and add_pr_options provide a means to work with such option sets, but modify them. They can be specified multiple times. To determine the effective option list, the options to add are collected and the option to remove are collected. From the option list determined by cm_options/pr_options or via cm_cfg/pr_cfg first all elements that are unifiable with an option to remove are removed and then the options to add are added.

Module craigtask_cm
Using the CM prover for interpolation tasks.
cm_craigproof(+Implication,+Options)
Compute Craig/Lyndon interpolants.
+Implication
A first-order implication.
+Options
A list of Key=Value pairs. The options are passed trough to predicates. In addition to (most of) the keys for cmpr the following keys are recognized:
debug
Specify a debugging level.
Number
If the number is > 2, then the validity of the input formula is checked with Prover9/Mace4, if it is > 3, then the interpolant property (w.r.t. predicates and constants) is, verified on the result with Prover9/Mace4. Default is 0.
enum_ips
Specify whether backtracking should be performed to search for alternate interpolants.
true
Perform backtracking.
false
Do not backtrack after an interpolant has been found (default).
goal_from_shared_signature
Specify clauses taking part in goal clause selection.
true
Only clauses containing a predicate shared by the interpolation sides take part in goal selection.
false
All clauses take part in goal selection (default).
ip
Return an interpolant.
-Variable
The variable will be bound to the computed interpolant, a first-order formula.
ip_dotgraph
Return a dotgraph representation of the underlying propositional proof tableau. See option tabx_dotgraph of cmpr/2.
ip_pref_side
Specify a side of the interpolation formulas which is preferred if coloring in both colors is possible.
a
The color of the left side (default).
b
The color of the right side.
ip_prop
Return the underlying propositional interpolant.
-Variable
The variable will be bound to the propositional interpolant (a propositional formula) that underlies the computed first-order interpolant.
ip_tabx
Return the underlying TABX tableau representation.
-Variable
The variable will be bound to the TABX tableau that underlies the computed propositional and first-order interpolants.

Module cm
Core module of the CM prover.
cm(+Matrix,?CompilationResult,+Options)
Compile the given matrix into a Prolog program.
+Matrix
A first-order matrix or a pair ?AbstractionTerm-+Matrix1 of a term and a first-order matrix. The pair format allows to extract variable bindings from instantiations of +Matrix1 made in proofs.
?CompilationResult
If it is a variable then it will be bound to the compliation result, a list of clauses, if it is an atom denoting a file name, then the compilation result is printed to the file.
+Options
A list of terms (CM options). The following terms are accepted:
acn
Supress cut after solution of propositional goals.
ff
Write compiled clauses with write_canonical instead of pretty printing.
hd
Depth measure: tableau depth.
hd1
Depth measure: combination of tableau depth and clause size.
hs
Depth measure: number of solved subgoals.
l
Sideways lemmas (leanCoP-like).
xlc
Cut after sideways lemma solution (incomplete, leanCoP-like).
mr2
ME-reduction with optimizing-only cut.
mrc
ME-reduction followed by cut (incomplete, leanCoP-like).
mrn
No ME-reduction.
no_fan
No fanning into contrapositive.
p
Construct proof term.
r1
Prune identical ancestors.
r4
Insert disequations that fail for subsumed clause instantiations (SETHEO-like).
r4_limit(+Number)
Inference limit for r4 before resorting to just unit subsumption.
r4a
Insert disequations that fail for subsumed clause instantiations (SETHEO-like). Restricted form that just considers unit subsumption.
r4a_max(+Number)
Maximal number of instantiations to consider for r4a.
r5
Insert disequations that fail for tautological clause instantiations (SETHEO-like).
r8(+Mode)
Antilemmas (SETHEO-line). The argument specifies what to compare: combinations of the subgoal with ancestors (al, a) and sideways lemmas (al, l), and subgoal only (v). Only al is in complete in general.
sg(+Mode)
Heuristic subgoal sorting: greedy.The argument is a number specifying the particular strategy. 0 is a standard strategy.
sn
No heuristic subgoal sorting.
sq(+Mode)
Heuristic subgoal sorting: quick and greedy, suggested for larger formulas.The argument is a number specifying the particular strategy. 0 is a standard strategy.
Remarks. From each the following option sets at most a single member should be given: [hd, hd1, hs], [mr2, mrc, mrn], [r4, r4a], [sg(_), sn, sq(_)]. For further options (including some experimental ones) see the source code.
proof(+GoalMatrix,+Options)
Process a previously loaded CM compilation result with respect to the given goal matrix.
+GoalMatrix
A first-order matrix.
+Options
A list of terms (PR options). The following terms are accepted:
d(+Number)
Initial depth for iterative deepening, default is 0.
depth_timeout(+Seconds)
If specified, perform iterative deepening with the given timeout at each depth.
i(+IncrementMode)
A number specifying the increment at iterative deepening (default 1) or a term specifying a particular incrementing strategy. Terms dyna/5 specify dynamic incrementing (see source code). The number 0 specifies increment by least required number.
p(-Variable)
Return the proof term. The variable will be bound to the proof term.
t(-Variable)
Return a list of used abstraction term instances.
